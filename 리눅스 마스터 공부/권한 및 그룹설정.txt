ls - l : 파일 속성 출력
- : 일반파일
d : 디렉터리 파일
p : 파이프
s : 소켓
b : 블록 장치 특수파일(하드디스크 등 저장장치)
c : 문자 장치 특수파일(마우스 등 입출력장치)
I : 기호적 링크·바로가기 역할(연결파일과 실제파일 위치 다름)

#소유권 명령어
1. chown : 파일·디렉터리 사용자·그룹 소유권 변경
ex) chown -R 소유자 그룹명 파일명
cf) 소유자, :그룹명
2. chgrp : 파일·디렉터리 그룹 소유권 변경
ex) chgrp -R 그룹명 파일명

#허가권 명령어
1. chmod : 파일·디렉터리 접근 허가권 변경
ex) chmod -R 파일명
chmod 666 파일명 -> rw rw rw
chmod -R o+w -> 기타 사용자에게 w권한 부여
2. umask : 새로 생성되는 파일·디렉터리 기본 허가 값 지정
cf) 기본권한 : 파일 666, 디렉터리 777
ex) umask 0002
-0664 -> -rw-rw-r--
d0775 -> drwxrwxr-x
ex) umask -S -> u=rwx, g=rwx, o=rx

#특수권한
1. SetUID(유저/s 4000), SetGID(그룹/s 2000)
-프로세스 실행되는 동안 해당 프로세스 root권한 임시로 가져옴
-사용자의 수준보다 높은 접근 요구시 해결방법
-권한 명시 디렉터리의 하위 디렉터리·파일도 영향
-실행파일 존재 : Setuid 소문자 s설정 -> 정상 실행
-실행파일 미존재 : setuid 대문자 S설정 -> 실행(X)
2. Sticky bit(/tmp 내 생성)
-공용디렉터리 사용 시 설정
-설정 디렉터리 내용 변경 : 파일 소유자 or root 한정( 타 사용자 파일 보호 목적)
-설정 디렉터리 접근 : 누구나 접근, 파일 생성 가능

#디스크 쿼터
-파일 시스템마다 파일 용량 및 개수 제한
-블록 단위 용량 제한, inode 개수 제한
-쿼터는 사용자별, 파일 시스템 별로 동작함
-웹호스팅 서비스에 유용

#디스크 쿼터 관련 명령어
1. quotaoff
-쿼터 서비스 비활성화
-a 정보출력, -u user 쿼터 비활성화, -g group 쿼터 비활성화, -v 메세지 출력
2. quotacheck
-파일 시스템의 디스크 사용 상태 검색
-a all, -u user, -g group, -m 재마운트 생략, -n 첫번째 검색된 것 사용, -p 처리결과 출력, -v 상태 출력
3. edquota
-편집기 이용하여 사용자나 그룹에 디스크 할당
-u user, -g group, -t 디스크 할당량 유예기간 설정, -p 디스크 할당량 설정을 타 사용자와 동일하게 설정
4. setquota
-명령행에서 직접 사용자나 그룹에 디스크 할당
-u user, -g group, -a all, -t 유예기간 초 단위 설정
